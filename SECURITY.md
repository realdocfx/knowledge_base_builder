# Security Policy

## Supported Versions

The following versions of Knowledge-Base-Builder are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2.0 | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Knowledge-Base-Builder, please report it responsibly. We appreciate your help in keeping the project and its users safe.

Please do **not** open a public issue for security-sensitive bugs. Instead, send a private message or email to the maintainer:

- GitHub: [realdocfx](https://github.com/realdocfx)

Include as much detail as possible:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The affected versions
- Any potential impact or proof-of-concept
- Suggested remediation, if known

We will respond as quickly as possible and coordinate a fix and disclosure.

## Security Best Practices

When using Knowledge-Base-Builder:

- Keep your system and dependencies up to date
- Only download content from trusted sources
- Be cautious with credentials and configure them using the official `ia configure` tool
- Verify downloaded files using the built-in checksum validation
- Report any suspicious behavior or potential vulnerabilities promptly

## Disclosure Policy

We follow a coordinated disclosure process. Once a fix is released, we will publish a security advisory and update the [CHANGELOG.md](CHANGELOG.md) accordingly.
